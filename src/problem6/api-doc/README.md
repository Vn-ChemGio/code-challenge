# 📘 API Documentation README

## 📖 Introduction
This document describes the APIs using the **OpenAPI (Swagger)** standard to help readers (especially Frontend Developers) understand how to interact with the Backend through clear request/response examples.

You can use this documentation to:
- Explore available APIs
- Understand request/response structures
- Quickly test APIs without a real backend

---

## 🚀 How to Use

### 1. Open Swagger Editor
Go to:
👉 https://editor.swagger.io/

### 2. Import API Doc
You can:
- Copy and paste the `.yaml` / `.json` content into the editor  
  **or**
- Upload the file directly

### 3. Try APIs
After importing:
- Swagger UI will display the list of APIs
- Select the endpoint you want to test
- Click **"Try it out"**
- Enter required parameters
- Click **"Execute"** to see the sample response

---

## 🧩 Document Structure
The API doc includes:
- **Paths**: List of endpoints
- **Methods**: GET, POST, PUT, DELETE, etc.
- **Parameters**: Query, Path, Body
- **Responses**: Mocked response data

---

## ⚠️ Important Notes

- This API documentation is **mocked**
- Main purposes:
    - Provide simple request/response examples
    - Help Frontend integrate easily with Backend in real scenarios

- Use https://editor.swagger.io/ to test this API doc

- In real projects:
    - API documentation is usually integrated directly into Backend code
    - Exposed in the **development (dev)** environment
    - Helps reduce manual work and keeps docs in sync with implementation
